# GoStudent Checkout UI - Technical Documentation

هذا المستند يقدم شرحاً تقنياً شاملاً لمشروع واجهة الدفع (Checkout UI) المصممة بأسلوب GoStudent. يغطي المستند الهيكلية، التقنيات المستخدمة، مكونات الواجهة، وكيفية تكامل المشروع مع ووردبريس.

## 1. التقنيات الأساسية (Tech Stack)

* **React 19:** مكتبة بناء واجهة المستخدم. تم استخدام المكونات الدالية (Functional Components) والـ Hooks (مثل `useState`, `useEffect`, `useMemo`).
* **Vite 8:** أداة بناء المشروع (Build Tool) الحديثة والسريعة، تستخدم بدلاً من Webpack.
* **React Hook Form (v7):** مكتبة لإدارة حالة النماذج (Forms) والتحقق من صحة البيانات (Validation) بأداء عالٍ وبدون إعادة تصيير (Re-renders) غير ضرورية.
* **React Country Flag:** لعرض أعلام الدول بجانب اختيار اللغة أو الدولة بصيغة SVG خفيفة.
* **CSS3 (Vanilla):** تم استخدام CSS خام بتصميم حديث يعتمد على المتغيرات (Custom Properties)، Grid، Flexbox، وخصائص الـ Logical Properties لدعم الـ RTL.
* **JSON Server:** خادم وهمي (Mock REST API) يستخدم في بيئة التطوير لمحاكاة قاعدة بيانات باستخدام ملف `db.json`.

## 3. شرح المكونات والصفحات (Components Breakdown)

التطبيق عبارة عن صفحة واحدة (Single Page Application) تتكون من عدة مكونات مجمعة في `App.jsx`.

### `App.jsx` (المنسق الرئيسي)
* **المسؤولية:** إدارة الحالة العامة للتطبيق (الأسعار المتاحة، الباقة المختارة، اللغة، اتجاه الصفحة RTL/LTR، والحسابات المالية).
* **العمليات:**
  1. يقوم بجلب خطط الأسعار عند بدء التشغيل عبر دالة `getPricing()`.
  2. يقوم بتغيير الـ `dir` والـ `lang` في وسم `<html>` بناءً على اللغة المختارة.
  3. يقوم بحساب الإجمالي (السعر الشهري، التخفيضات، الإجمالي النهائي) ويمرر هذه الحسابات للمكونات الفرعية.

### `CheckoutForm.jsx` (نموذج الدفع)
* **المسؤولية:** عرض بطاقات الأسعار، وجمع بيانات المستخدم (الاسم، الإيميل، بيانات الدفع).
* **التقنيات:** 
  * يستخدم `useForm` للتحكم في الحقول والتحقق من صحتها (Validation).
  * يحتوي على دعم كامل لتعدد اللغات (i18n) داخلياً لترجمة النصوص.
  * يرسل البيانات للخادم عند نجاح التحقق عبر دالة `submitOrder()`.

### `OrderSummary.jsx` (ملخص الطلب)
* **المسؤولية:** عرض ملخص مالي للحسابات التي تمت في `App.jsx`.
* **المميزات:** شريط جانبي ثابت (Sticky) يعرض سعر الجلسة، الإجمالي الشهري، قيمة الخصم (إن وجد)، والسعر النهائي. يحتوي أيضاً على مكون `TrustBadges`.

### `SuccessModal.jsx` (نافذة النجاح)
* **المسؤولية:** نافذة منبثقة (Modal) تظهر فوق الشاشة بخلفية ضبابية بعد نجاح إرسال الطلب. تحتوي على حركة (Animation) لعلامة الصح.

### `LoadingSkeleton.jsx` (هيكل التحميل)
* **المسؤولية:** تحسين تجربة المستخدم (UX) عبر عرض "هيكل وهمي" للمحتوى متحرك (Shimmer effect) أثناء انتظار رد الـ API بدلاً من شاشة بيضاء.

### `TrustBadges.jsx` (شارات الثقة)
* **المسؤولية:** عرض أيقونات SVG (مثل دفع آمن، ضمان استرجاع) لزيادة موثوقية صفحة الدفع.

## 4. تصميم واجهة المستخدم (UI/CSS Architecture)

التصميم يعتمد على ملفين أساسيين في مجلد `src/styles/`:

1. **`global.css` (نظام التصميم):**
   * يحتوي على الـ Design Tokens كمتغيرات (مثال: `--color-primary`, `--space-md`).
   * يعمل على تصفير التنسيقات الافتراضية للمتصفحات (Reset).
   * يطبق الـ Glassmorphism (تأثير الزجاج الشفاف) في المتغيرات.
   * يحتوي على حركة (Animation) لخلفية الصفحة (Gradient Background Shift).

2. **`app.css` (تنسيق المكونات):**
   * يستخدم **CSS Logical Properties** (مثل `margin-inline-start` بدلاً من `margin-left`). هذا هو السر التقني الذي جعل الواجهة تنعكس بشكل مثالي من اليمين لليسار (RTL) عند اختيار اللغة العربية، دون الحاجة لكتابة كود CSS مكرر أو خاص بالـ RTL.
   * يستخدم CSS Grid لبناء تخطيط متجاوب (Responsive) يتغير من عمود واحد في الموبايل إلى عمودين في الشاشات الكبيرة.
   * يطبق حركات (Micro-animations) مثل الاهتزاز `shake` عند خطأ الإدخال.

## 5. طبقة البيانات والاتصال (API & Services)

ملف `src/services/api.js` هو المسؤول عن الاتصال. تم تصميمه بذكاء ليعمل في **ثلاث سيناريوهات**:

1. **بيئة التطوير (JSON Server):** يتصل بالرابط `http://localhost:3001` ويقرأ/يكتب في ملف `db.json`.
2. **بيئة الووردبريس (WordPress REST API):** يتحقق إذا كان هناك كائن `window.gostudentApiSettings` (يتم حقنه من الووردبريس)، وإذا وجده يستخدم الروابط والـ Nonce الخاص بالووردبريس.
3. **وضع الـ Fallback (عدم وجود خادم):** إذا فشل الاتصال بالخادم، يقوم آلياً بإرجاع بيانات وهمية (Mock Data) موجودة داخل الكود، ويقوم بمحاكاة تأخير الشبكة (Artificial Latency) حتى يرى المستخدم تأثيرات التحميل، ويسمح للواجهة بالعمل دون أي أخطاء (Demo Mode).

## 6. التكامل مع ووردبريس (WordPress Integration)

لجعل مشروع React يعمل داخل ووردبريس، اتبعنا الخطوات التالية:

### الخطوة 1: تعديل أداة البناء Vite (`vite.config.js`)
* تم تعيين `base: ""` لجعل مسارات الملفات المخرجة نسبية (Relative).
* تم إيقاف تقسيم ملفات الـ CSS (`cssCodeSplit: false`) ليخرج ملف CSS واحد يسهل استدعاؤه.
* تم إجبار Vite على إخراج الملفات بأسماء ثابتة (`index.js` و `style.css`) بدلاً من إضافة رموز عشوائية (Hashes) ليسهل ربطها بالـ PHP.

### الخطوة 2: ملف الإضافة (Plugin File)
تم إنشاء ملف `wordpress/gostudent-order.php` بلغة PHP:
* **تعريف الإضافة:** ليتعرف عليها ووردبريس كـ Plugin.
* **إضافة Shortcode:** يقوم الدالة `render_shortcode` بطباعة `<div id="root"></div>` وهو المكان الذي سيعمل فيه React.
* **استدعاء الملفات (Enqueue):** تستخدم دالة `wp_enqueue_script` لجلب ملف `index.js` و `wp_enqueue_style` لجلب ملف `style.css` من مجلد `dist/assets/`.
* **تصدير المتغيرات (Localize):** باستخدام `wp_localize_script`، نرسل رابط الـ API الخاص بووردبريس وتصريح الأمان (Nonce) إلى الجافاسكريبت.
* **تسجيل الـ Endpoints:** تم تسجيل مسارين في الـ WP REST API (`/gostudent/v1/pricing` لجلب الأسعار، و `/gostudent/v1/orders` لاستقبال الطلب).

## 7. خطوات البناء والتشغيل (How to Run/Build)

1. **للتطوير المحلي (Development):**
   ```bash
   npm run dev:full
   ```
   يقوم هذا الأمر بتشغيل الـ React على بورت 5173، والـ JSON Server على بورت 3001 معاً بفضل مكتبة `concurrently`.

2. **للبناء للإنتاج (Production/WordPress):**
   ```bash
   npm run build
   ```
   يقوم Vite بضغط الكود، تحويل الـ ES6 إلى كود متوافق مع المتصفحات، واستخراج الملفات النهائية إلى مجلد `dist/`.

3. **للووردبريس:**
   يتم أخذ مجلد `dist/` وملف `gostudent-order.php` ووضعهم في مجلد واحد داخل `wp-content/plugins/` في موقع الووردبريس.

---

### ملخص الإنجازات التقنية في المشروع:
- **أداء عالي:** لا يوجد إعادة تصيير (Re-renders) للنموذج بفضل `react-hook-form`.
- **أكواد مرنة:** دعم الـ RTL لم يتطلب أي مكاتب خارجية، فقط CSS حديث.
- **تأقلم ذكي (Resilience):** الواجهة تعمل في حالة فشل الخادم ولن تنهار أبداً.
- **جاهزية الإنتاج:** ملفات جاهزة للعمل كإضافة ووردبريس أو كصفحة ثابتة (Static Page).
